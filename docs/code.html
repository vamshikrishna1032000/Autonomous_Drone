Drone Control Code Documentation

REQUIREMENTS

  1. A conda environment based upon the libraries listed in environment.yml
  2. AirSim installed (from Colosseum) and placed in the Plugins directory of an Unreal Engine map.
  3. ZED SDK installed
  4. (Optional) ROS Environment initialized.

follow_object_airsim.py

Connects the ZED camera on a laptop to an AirSim client. It employs an object detector network with a single detection category as humans, then captures the first person, reads the distance, and then directs the drone to move towards that distance in meters. The y-axis is fixed in this demo to prevent descending towards the ground.

airsim_toROS.py

This module converts the AirSim API to ROS interpretable commands.

follow_path_airsim.py

Experimental code for interpreting a predefined map for the AirSim drone to follow(Still in development!).

modules.py

A set of simple AirSim flight methods that work with or without a ROS connection (see line 15 for details). It provides basic functionalities like connecting to the drone, moving in specified directions, and taking off.

testFlight.py

A simple test flight script to confirm compatibility with the setup and ensure that all components are functioning as expected. For relevant documentation (inherits all methods from modules.py)
